# 后续开发路径规划

> **执行规则**：本文件是 Plan1 完成后的阶段规划入口。每完成一个阶段，必须先本地验证，再使用中文 commit message 提交，并立即 push 到 GitHub。

**目标**：把当前高保真 Mock Demo 推进为可联调、可验收、可上线的微信小程序商城。

**当前基线**：Plan1 Demo Foundation 已完成；小程序前端 35 页面可演示；后端已有 Spring Boot 骨架、商品基础 CRUD、Mock 接口和 Docker 配置。当前主要缺口是真实登录、真实交易闭环、会员营销、管理后台、联调上线。

**总体策略**：先打通真实交易闭环，再补会员营销和用户体验，最后做管理后台与上线联调。所有阶段都以可验证结果为出口，不做没有验收标准的泛化开发。

## 阶段提交规则

每个阶段完成后执行：

```bash
git status --short --branch
git add <本阶段变更文件>
git commit -m "docs: 更新阶段规划"

```

开发阶段 commit message 使用中文，例如：

```bash
git commit -m "feat: 完成真实登录与令牌刷新"

git commit -m "fix: 修复订单提交金额计算"
```

如本机直连 GitHub 失败，使用已验证过的代理参数：

```bash
git -c http.proxy=http://127.0.0.1:33210 -c https.proxy=http://127.0.0.1:33210 -c http.sslBackend=openssl push origin main
```

## Phase 1：Plan2 核心交易闭环

**目标**：用户可以使用真实后端完成「登录 -> 浏览商品 -> 加入购物车 -> 提交订单 -> 支付适配 -> 查看订单」。

**详细计划文件**：`docs/superpowers/plans/2026-07-16-plan2-core-trade-loop.md`

**子阶段**：
1. 登录与会话
   - 接入微信登录 code2session 的服务边界。
   - 完成用户创建、openid 绑定、JWT 生成、Token 刷新。
   - 前端请求从 mock token 切换为真实 token。
2. 商品真实接口
   - 商品分类、商品列表、商品详情从 `MockData` 切换到数据库查询。
   - 保留演示数据 seed，便于本地初始化。
   - 补齐 SKU、库存、收藏接口的最小可用版本。
3. 购物车真实接口
   - 新增 `trade_cart` 数据模型、Mapper、Service、Controller。
   - 支持加购、列表、改数量、删除、选中、购物车数量统计。
   - 前端 `cart/*` 接口切换真实后端。
4. 地址与结算
   - 新增收货地址 CRUD。
   - 结算页支持默认地址、商品明细、运费、优惠占位。
   - 运费先实现固定运费和满额包邮。
5. 订单模块
   - 新增订单主表、订单明细表。
   - 支持提交订单、订单列表、订单详情、取消订单、确认收货。
   - 完成库存扣减和未支付取消后的库存回补。
6. 支付适配
   - 继续保留 Mock 支付用于本地联调。
   - 后端建立支付单模型和微信支付 V3 配置入口。
   - 商户号未就绪时，明确返回 mock 支付参数；商户号就绪后切换真实支付。

**验收标准**：
- 后端 `mvn install -DskipTests` 通过。
- 前端 `useMock: false` 时，首页、分类、详情、购物车、结算、订单页面能跑通。
- 用户至少能完成一次完整的本地 Mock 支付订单。
- 阶段完成后中文 commit 并 push。

## Phase 2：会员与营销体系

**目标**：建立付费会员、优惠券、满减、会员价、分享奖励的最小可用闭环。

**计划文件**：待 Phase 1 完成后创建。

**子阶段**：
1. 会员方案与会员身份
   - 月卡、年卡方案配置。
   - 会员开通、续费、到期时间、会员价展示。
2. 优惠券
   - 券模板、领券、我的优惠券、下单用券。
   - 新人券先做固定规则。
3. 满减与价格计算
   - 满 X 减 Y。
   - 价格计算顺序：商品价 -> 会员优惠 -> 满减 -> 优惠券 -> 运费。
4. 分享奖励
   - 分享参数、推荐关系绑定、订单完成后奖励入账。
   - 提现先做申请记录，不直接接企业付款。

**验收标准**：
- 下单金额能正确应用会员价、满减、优惠券。
- 会员中心不再提示“开发中”。
- 分销中心有收益统计与推广入口。
- 阶段完成后中文 commit 并 push。

## Phase 3：用户体验补全

**目标**：补齐售后、评价、物流展示、消息、钱包等用户侧 P0 体验。

**子阶段**：
1. 售后退款
   - 申请售后、商家审核、售后状态流转。
2. 商品评价
   - 订单完成后评价、商品详情展示评价。
3. 物流展示
   - 首期手填物流公司和单号，用户端展示。
   - 不接第三方物流 API。
4. 消息与红点
   - 订单支付、发货、售后结果的站内消息。
5. 钱包与余额明细
   - 分销奖励入账、余额明细、提现申请记录。

**验收标准**：
- 前端不再出现“退款申请功能开发中 / 物流查询功能开发中 / 评价功能开发中”的提示。
- 订单完成后的售后与评价路径可用。
- 阶段完成后中文 commit 并 push。

## Phase 4：管理后台

**目标**：运营人员可以通过 PC 后台管理商品、订单、用户、营销和内容。

**子阶段**：
1. 后台脚手架
   - Vue 3 + TypeScript + Vite + Element Plus。
   - 登录、路由、布局、接口封装。
2. 商品与分类管理
   - 商品增删改查、上下架、分类管理。
3. 订单管理
   - 订单列表、详情、发货、取消、售后审核。
4. 会员与营销管理
   - 会员方案、优惠券模板、满减活动。
5. 内容管理
   - Banner、公告、首页推荐位。

**验收标准**：
- 运营可以独立录入商品并在小程序端看到。
- 运营可以处理订单发货。
- 阶段完成后中文 commit 并 push。

## Phase 5：联调、安全与上线

**目标**：完成真机联调、微信支付、云托管部署、安全加固和提审准备。

**子阶段**：
1. 微信支付真实联调
   - 商户号、证书、APIv3 密钥、回调验签。
   - 真实 1 分钱支付测试。
2. 云托管部署
   - 后端 Docker 构建。
   - 云托管 MySQL、Redis、环境变量配置。
3. 真机测试
   - iOS、Android、不同微信版本。
   - 检查登录、下单、支付、订单、会员、售后。
4. 安全加固
   - 关键接口鉴权、频率限制、敏感信息脱敏、配置外置。
5. 提审准备
   - 小程序类目、隐私协议、资质材料、体验账号。

**验收标准**：
- 云环境健康检查通过。
- 真机完整交易闭环通过。
- 提审材料完整。
- 阶段完成后中文 commit 并 push。
